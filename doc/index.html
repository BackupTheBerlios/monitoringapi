 Description
MAPI can be broken down into two major components.
  • Linux kernel modules.
Linux kernel modules extends the functionality of PF_PACKET protocol family. To be more specific MAPI gives the ability to apply functions to packets. Such functions can do many things such as
  • Count packets.
  • Count bytes.
  • Log packets.
  • Search for a string.
  • Reassemble IP fragments.
  • Reconstruct UDP/IP or TCP/IP streams.
  • Sample packets.
  • User space library.
Gives a friendly interface to users of MAPI. So the user is not supposed to know the ioctls used by MAPI etc.

The idea behind MAPI is that the network monitoring application will use some of the predefined functions and will not require to copy all packets to user space. So a great performance advantage will be gained.If such a function does not exist application can load  its own if this is permitted.

MAPI gives an interface with which applications can assign jobs, remove jobs, read results and read packets. Also, as already written MAPI gives an interface which leverages the loading of new functions.
